Release Go 1.27 RC1
2026-06-18 · go1.27rc1
Go 1.27 Release Candidate 1 is out. Includes experimental SIMD support under GOEXPERIMENT=simd, portable size-agnostic simd package, updated archsimd API for AMD64, and archsimd support for Wasm and ARM64. Some breaking API changes for AMD64 archsimd.
Accepted Landlock Process Sandboxing for Linux
A 2024 proposal for Go to support process sandboxing on Linux using Landlock LSM has been accepted. Will allow Go programs to restrict their own filesystem access using Landlock rules.
Accepted simd/archsimd: Architecture-Specific SIMD Intrinsics (GOEXPERIMENT) — #73787 ★349
The umbrella proposal for architecture-specific SIMD intrinsics under a GOEXPERIMENT. Go 1.27 RC1 includes the first implementation for AMD64, ARM64, and Wasm.
Accepted simd: Architecture & Vector-Size Agnostic SIMD Intrinsics — #78902
Second-level SIMD API providing a portable interface to SIMD vectors not tied to a particular architecture or vector length. Auto-selects between AVX, AVX2, AVX512. Supported vector methods are the intersection of Wasm SIMD API and current AMD64 SIMD.
Closed flag: add All, AllSet iterators — #65675
Proposal to add All() iter.Seq[*Flag] and AllSet() iter.Seq[*Flag] to the flag package, making flag iteration compatible with range-func. Closed with positive reception.
Closed sort: add SearchWithError — #74384
Adds sort.SearchWithError allowing the search function to return an error and exit early. PR at golang/go#74372. Frozen due to age.
Closed x/sync/singleflight: make Group.Do generic — #80120
Proposal to make singleflight.Group.Do generic: Do[T any](key string, fn func() (T, error)) (T, error, bool). Commenters noted this is a breaking change.
Closed errors: add All and AllAs iterators — #66455
Proposal to add iterators for walking the error tree, similar to errors.Is/As but in a single pass. Frozen due to age.
Closed x/net/http2: deprecate ClientConnPool — #77695
Part of the x/net/http2 → std migration (#67810). ClientConnPool has been superseded by the client connection API from #75772.
Open net/http: add MethodQuery constant — #80058 ★11
RFC 10008 defines the HTTP QUERY method. Proposal to add net/http.MethodQuery constant, following the same pattern as MethodPatch.
Open spec: interface literals — #25860 ★93
Long-standing proposal for interface literals in the Go spec. Still under discussion.
Open all: add wasip3/wasm port — #77141 ★60
Proposal to add a wasip3/wasm port to Go, following wasip1.

CVE-2026-46602 x/image/tiff: Lack of Tile Size Limits
Public track · Fixed
TIFF images may have tiles larger than the image size. x/image/tiff didn't validate tile sizes, allowing malicious input to consume excessive memory or crash. Fixed.
x/image/vp8l: Excessive Memory Allocation
Open
VP8L images with unused Huffman tree groups can cause ~175MiB slice allocations (up to ~1.4GiB with group data). A small malicious image can trigger this.
html/template: iframe srcdoc Not Properly Escaped
Open · NeedsFix
Actions placed in iframe srcdoc attributes are not properly treated as HTML and escaped. Treated as security hardening issue due to low likelihood of exploitation.
net/smtp: STARTTLS Case-Sensitivity Downgrade
Open · NeedsFix
RFC 5321 requires EHLO keywords to be compared case-insensitively. net/smtp's STARTTLS handling is case-sensitive, allowing a downgrade attack.
crypto/tls: Omit PSK in ECH Outer Client Hello
Backport to 1.25, 1.26
Including the PSK extension in ECH outer Client Hello allows an on-path attacker to harvest and replay hellos with guessed SNI values. Fixed in master, backporting to 1.25 and 1.26.
